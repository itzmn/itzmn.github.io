Spark基础概念

发表于 2019-08-15 | 分类于 java ， 大数据 ， Spark ， Scala
字数统计: 629 字 | 阅读时长 ≈ 2 分钟
了解Spark的基本概念

作用

Apache Spark 是一种快速的，通用的，可扩展的大数据分析引擎。是基于内存的计算

Spark架构

Driver是用户程序，cluster manager是集群资源管理器（每种不同的部署模式有不同的集群管理器）, Worker Node是真正运行程序的机器节点，Executor执行程序的JVM进程

大致运行过程:

  1. 用户创建sparkContext时，SparkContext会连接到Cluster Manager。Cluster Manager会根据用户提交程序时设置的参数封装CPU，内存分配资源，启动Executor
  2. Driver程序会将Spark程序划分为不同的Job，每个Job划分为不同的Stage，每个Stage有一组Task，Task会被分配到Executor节点中执行
  3. Executor接收到Task之后，会下载Task执行的数据依赖，准备好之后，开始执行，并将执行状态汇报给Driver
  4. Task分两种，ShuffleMapTask，用来做数据洗牌，ResultTask负责生成结果
  5. Driver会不断的调用Task，将Task发送到Executor执行，直到所有的Task执行完毕

Application

应用，用户使用Spark编写的程序

Driver

SparkContext所在的节点位置

Container

封装了CPU，内存，Core等资源的容器

RDD

分布式弹性数据集。具有自动容错，位置感知性调度和可伸缩性等特点。生成RDD的方式一种是读取数据生成RDD，一种是从RDD转换而来

RDD操作（算子）

分为动作算子，转换算子

转换算子将Parent RDD 转换称为新的RDD，这是一系列的逻辑

动作算子是计算RDD的数据，RDD是懒计算的，当遇到动作算子，RDD才会真正的开始计算

Partition

Spark一般使用HDFS作为文件存储的地方，HDFS文件存储会划分为Block存储在不同的节点上，Spark读取文件生成RDD，RDD的Partition就是Block的个数

Executor

执行Task的JVM

Job

当Spark调用了动作算子，生成一个Job

Dependency

依赖，RDD要么通过读取数据而来，要么通过转换而来，通过转换来的RDD都是Parent RDD的概念，是如何从Parent RDD之间取得数据形成新的RDD的，这个就是依赖关系

Stage

依赖分为宽依赖可窄依赖，当RDD之间是宽依赖的时候，就会划分为一个Stage

Task

在一个Stage中，RDD的每个Partition的计算过程是相同的，每个Partition是相互隔离的，可以将每个Partition打包封装成Task发送到节点运行

Shuffle

当RDD之间是宽依赖的时候，就会产生Shuffle
