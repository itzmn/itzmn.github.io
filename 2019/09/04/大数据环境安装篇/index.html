简介

为了更好的学习大数据，我们自己安装大数据环境，写下笔记为了复习，这里安装的是Apache版本的环境，需要手动配置配置文件，后面如果有需要可以搞一下CDH版本的

需要安装哪些

  1. java
  2. scala
  3. hadoop
  4. spark
  5. zookeeper
  6. hive
  7. mysql
  8. kafka

本文共搭建了3台虚拟机，预计安装如下

ip 192.168.33.3 192.168.33.4 192.168.33.5
机器名 hadoop1 hadoop2 hadoop3
hdfs namenode and datanode datanode datanode and sencodenamenode
yarn nodemanager resourcemanager nodemanager

虚拟机

大数据环境，肯定要有几台主机，就用虚拟机来替代，上一篇已经教过如何安装虚拟机并配置网络了，如有不懂，看上一篇

https://blog.itzmn.com/2019/09/02/%E5%A4%A7%E6%95%B0%E6%8D%AE%E5%BC%80%E5%8F%91%E7%8E%AF%E5%A2%83%E5%AE%89%E8%A3%85%E4%B9%8B%E8%99%9A%E6%8B%9F%E6%9C%BA%E5%AE%89%E8%A3%85%E9%85%8D%E7%BD%AE/

  1. 安装完虚拟机后，配置ip，主机名

  2. 配置机器之间免密登录

JDK

  1. 官网下载jdk安装包上传到虚拟机，解压

  2. 配置环境变量 修改 /etc/profile 文件

  3. 重新加载环境变量 source /etc/profile

Scala

  1. 官网下载scala安装包上传到虚拟机，解压

  2. 配置环境变量 修改 /etc/profile 文件

  3. 重新加载环境变量 source /etc/profile

Hadoop

  1. 上传压缩包，解压

  2. 配置环境变量

  3. 修改hadoop解压包下etc/hadoop/hadoop-env.sh配置java环境

  4. 配置hadoop相关的配置文件

    1. core-site.xml

    2. hdfs相关

      1. hadoop-env.sh

      2. hdfs-site.xml

    3. yarn 相关

      1. yarn-env.sh

      2. yarn-site.xml

    4. MapReduce相关

      1. mapred-env.sh

      2. mapred-site.xml

启动Hadoop集群

  1. 第一次启动需要初始化namenode

  2. 启动hdfs(在namenode节点机器启动)

  3. 启动yarn(在resourcemanager节点机器启动)

Zookeeper

  1. 上传压缩包，解压

  2. 在conf目录下复制zoo.cfg

  3. 修改zoo.cfg

  4. 在zoo.cfg文件中有配置zookeeper的数据目录。在数据目录中新建myid文件，内容为上面的server.id的id

Mysql

Hive

  1. 上传安装包，解压

  2. hive-env.sh

  3. hive-site.xml

  4. hive-log4j.properties

Kafka

  1. 上传压缩包，解压

  2. 修改server.properties

Spark

spark on yarn 模式，当我们Hadoop集群搭建完毕之后，只需要简单配置即可

  1. 下载安装包，解压

  2. 配置spark-env.sh
